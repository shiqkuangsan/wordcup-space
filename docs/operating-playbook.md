# 操作手册

## 目标

使用 Codex 分析每场世界杯比赛，辅助你判断喜欢的球队是否值得买，并记录 User 和 Codex 两套决策来源。真实平台账户可以只有一个；系统内部通过决策来源、执行人、资金归属和真实资金标记来区分。

## 双主体记录

系统必须同时记录用户投入和 Codex 决策，并在统计时明确区分：

| 主体 | decision_by | 用途 |
|---|---|---|
| 用户 | `user` | 记录你自己的判断、投入、个人复盘。 |
| Codex | `codex` | 记录 Codex 的模型分析、自主决策、额度使用和复盘。 |

关键原则：

- 所有预测、持仓、台账和复盘都要记录 `decision_by`。
- 所有实际下单记录都要记录 `placed_by`；默认都是 `user`，表示你代为执行。
- 真实平台账户可以只有一个，用 `platform_account` 记录，默认使用 `betway-main`。
- 涉及资金的记录必须归属到明确的 `portfolio_id`，例如 `user` 或 `codex`。
- 每张注单都要记录 `is_real_money`，用于区分真实资金和模拟记录。
- 同一场比赛可以同时存在用户决策和 Codex 决策，不能互相覆盖。
- dashboard 默认同时展示两套结果，也要支持单独过滤用户或 Codex。
- Codex 可以参考用户提供的数据，但必须标记数据来源；用户自己的投入不能混入 Codex 的额度统计。
- Codex 的真实资金单，本质是 `decision_by=codex`、`placed_by=user`、`is_real_money=true`。

## 单场比赛流程

每场比赛按下面状态推进：

| 状态 | 含义 |
|---|---|
| `scheduled` | 已有赛程，但还没有正式分析。 |
| `priced` | 至少记录了一组博彩公司赔率快照。 |
| `analyzed` | 已记录模型概率和定性分析。 |
| `decided` | 已做决策：下注、放弃、观察，或作为串关一腿。 |
| `settled` | 已记录赛果和资金影响。 |
| `reviewed` | 已完成赛后决策质量复盘。 |

## 预测流程

第一版默认流程：

1. 把博彩公司赔率换算成隐含概率，并去除水位/抽水影响。
2. 用最优可得赔率建立市场基准概率。
3. 根据球队实力、近期状态、伤停、首发、场地/旅行/休息、战术对位、赛事背景做调整。
4. 对支持的市场输出概率：`1X2`、让球/亚洲盘口、大小球；数据质量足够时再考虑部分特殊玩法。
5. 比较模型概率和当前盘口赔率。
6. 按资金规则决定下注金额。
7. 开赛前记录决策。
8. 赛后记录结果、收盘线价值和复盘。

## 资金规则

Codex 可配置独立额度。初始建议额度约 `1000`，以你实际分配为准。

| 规则 | 默认值 |
|---|---:|
| 普通单场下注上限 | 资金的 `10%` |
| 高信心单场下注上限 | 资金的 `20%` |
| 串关下注上限 | 资金的 `5%` |
| 单日亏损上限 | 资金的 `40%` |
| 出手最低优势 | `3%` 期望值 |
| 自主下注最低数据质量 | `medium` |

不追损。连续亏损时降低风险，而不是加大下注。

## 决策模式

| 模式 | 目的 | 规则 |
|---|---|---|
| `support` | 帮你判断是否支持喜欢的球队 | 重点说明风险、公允赔率、情绪溢价和替代选择。 |
| `autonomous` | Codex 自主选择模拟下注 | 必须遵守资金规则并记录理由。 |
| `parlay` | 模拟串关 | 只允许小额，并记录相关性风险。 |
| `pass` | 不下注 | 放弃也是有效决策，也要计入统计。 |

## 资金账本

| 账本 | 归属 | 说明 |
|---|---|---|
| `user` | 用户 | 记录你的判断、投入、真实/模拟决策。 |
| `codex` | Codex | 记录 Codex 的自主决策和额度使用，可包含真实资金或模拟记录。 |

用户账本和 Codex 账本分别计算余额、盈亏、ROI、回撤和风险敞口。跨账本只做对比，不做合并收益。

如果某条记录只是“用户提供盘口给 Codex 参考”，它属于数据来源，不自动算作用户下注。

如果你用同一个平台账户代 Codex 下单，记录方式是：

```text
platform_account = betway-main
portfolio_id = codex
decision_by = codex
placed_by = user
is_real_money = true
```

## 可视化目标

- 资金曲线和回撤。
- ROI、yield、命中率、平均赔率。
- User vs Codex 的收益、命中率、赔率质量、收盘线价值对比。
- 按球队、小组、市场、博彩公司统计风险敞口。
- 模型校准：预测概率区间 vs 实际命中率。
- 按市场和博彩公司统计收盘线价值。
- 单场比赛卡片：赛前理由和赛后复盘。
- 串关树和相关性说明。

## AI 使用规则

AI 可以总结新闻、比较球队、审查推理、起草比赛笔记。不能编造数据。每个数据型判断都要引用已记录来源，否则标记为假设。

Codex 不需要把强 AI 能力内置到 app。默认策略是：

- app 负责可靠记录：赛程、盘口、决策、执行、注单、结算和资金账本。
- Codex 负责研究和解释：必要时调用 agent skills、公开网页或本地 provider 数据，再把结果整理成结构化理由。
- `machina-sports/sports-skills` 只作为 Codex/agent 工具安装，不作为 Next.js 运行时依赖。
- `rezarahiminia/worldcup2026` 只作为赛程/球队/场馆/比分 provider，不直接生成下注建议。
- 所有 Codex 决策默认先走 `dryRun` 或人工确认；外部工具不得绕过 `/api/intents`、`/api/bet-slips` 和风控规则。
- 市场价格、预测市场和新闻情绪只能作为参考信号，不能写成确定性结论。

## 当前支持玩法

- 全场/半场：胜平负、让球、大小球。
- 串关：最多 7 legs，默认小额风控。
- 特殊玩法：第 N 个进球是哪只队伍。
- 结算支持：赢、输、走水、半赢、半输、提前兑现、取消/无效。

默认资金都是真实资金；只有明确选择“模拟记录”或 API 传 `isRealMoney=false` 才作为模拟。
