# 世界杯下注工作台流程

## 使用目标

- 记录 User 与 Codex 两套决策来源。
- 默认都是真实资金，只有明确标记为模拟时才是模拟记录。
- Codex 可以给建议，也可以在可行时通过 Chrome / Computer Use 操作；但只有确认下单成功后才生成注单并扣款。

## 主流程

```mermaid
flowchart TD
  A["同步真实赛程"] --> B["今日/近期比赛池"]
  B --> C["录入或读取盘口赔率"]
  C --> D{"决策来源"}
  D -- "User" --> E["记录 User 决策"]
  D -- "Codex" --> F["Codex 分析并给出建议"]
  F --> G{"执行动作"}
  E --> G
  G -- "放弃/等待" --> H["记录不下注原因"]
  G -- "用户代下单" --> I["执行 Attempt"]
  G -- "Codex 操作 Chrome/Computer Use" --> I
  I --> J{"是否成功成交"}
  J -- "否" --> K["记录失败/取消/赔率变化，不扣款"]
  J -- "赔率变化 >= 6%" --> L["回到决策复核"]
  J -- "是" --> M["生成 Bet Slip，扣对应账本资金"]
  M --> N["注单中心：未结算池"]
  N --> O["平台结果/比分/截图同步"]
  O --> P{"结算结果"}
  P -- "赢" --> Q["返还 stake * odds"]
  P -- "输" --> R["不再返还"]
  P -- "走水/取消" --> S["返还本金"]
  P -- "半赢" --> T["返还本金 + 半注盈利"]
  P -- "半输" --> U["返还半注本金"]
  P -- "提前兑现" --> V["按平台兑现额返还"]
  Q --> W["资金账本"]
  R --> W
  S --> W
  T --> W
  U --> W
  V --> W
  W --> X["复盘：决策质量、执行质量、盘口质量"]
```

## 支持玩法

| 维度 | 当前支持 |
|---|---|
| 时间段 | 全场、半场 |
| 市场 | 胜平负、让球、大小球、第 N 个进球球队、串关 |
| 结算 | 赢、输、走水、半赢、半输、提前兑现、取消/无效 |

## 信息不完整时必须提示

创建或结算记录前，如果缺少这些信息，需要提示用户补充或标记为未知：

- 比赛：哪一场，或至少双方球队和开球时间。
- 市场：全场/半场，胜平负/让球/大小球/第 N 球/串关。
- 选择：买哪一边，例如阿根廷胜、大 2.5、第 1 球巴西。
- 金额、赔率、平台账户。
- 是否真实资金；默认真实。
- 平台注单号或截图备注。
- 结算依据：平台已结算、比分来源、截图或用户口述。

## 下注成功前不扣款

`bet_intent` 和 `execution_attempt` 都不改变资金。只有确认成交后生成 `bet_slip` 才扣款。
